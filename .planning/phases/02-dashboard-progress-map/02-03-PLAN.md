Status sort priority (locked decision from CONTEXT.md):
```typescript
type MilestoneStatus = "not_started" | "in_progress" | "complete";
const NEXT_STATUS: Record<MilestoneStatus, MilestoneStatus> = {
  not_started: "in_progress",
  in_progress: "complete",
  complete: "not_started",
};
const STATUS_PRIORITY: Record<MilestoneStatus, number> = {
  in_progress: 0,
  not_started: 1,
  complete: 2,
};
// Sort the INITIAL prop array in MilestoneGrid — do NOT re-sort optimistic state
// (prevents jarring card jumps on badge click — see Research pitfall 5)
```
